APPROVED RULE FUNCTIONS AND THEIR PARAMETERS

── DQ Functions ──────────────────────────────────────

Completeness
  RuleType: Completeness
  RuleParameters: """"
  Logic: fails if field is null, empty, whitespace, or equals -99999 (system null sentinel).

ValidityRange
  RuleType: Validity
  RuleParameters: {""MinRange"": <number>, ""MaxRange"": <number>}
  Logic: fails if value < MinRange or > MaxRange.

CurveSpikes
  RuleType: Validity
  RuleParameters: {""WindowSize"": 5, ""SeveritySize"": 4}   ← omit for defaults
  Logic: detects statistical spikes in log curve arrays (MEASURED_VALUE field).

IsNumber
  RuleType: Validity
  RuleParameters: """"
  Logic: fails if field is not a pure integer (regex ^[0-9]+$). No decimals.

StringLength
  RuleType: Validity
  RuleParameters: {""Min"": <number>, ""Max"": <number>}
  Logic: trims then checks length. Set Min=Max for exact-length requirement.

IsEqualTo
  RuleType: Validity
  RuleParameters: {""Value"": ""VAL1,VAL2"", ""Delimiter"": "",""}
  Logic: fails unless field matches one of the delimited values (trimmed). Delimiter defaults to comma.

IsGreaterThan
  RuleType: Validity
  RuleParameters: {""Value"": <number>}
  Logic: fails if field < Value.

Entirety
  RuleType: Entirety
  RuleParameters: {""DataType"": ""<childType>"", ""Name"": ""<childName>""}
  Logic: fails if no child record with that name exists under the parent index node.

Consistency
  RuleType: Consistency
  RuleParameters: """"
  Logic: compares DataAttribute value against the same field in a reference database.

Uniqueness
  RuleType: Uniqueness
  RuleParameters: """"
  Logic: fails if another record shares the same UNIQKEY in the index.

── Prediction Functions ──────────────────────────────

DeleteDataObject
  SaveType: Delete
  RuleParameters: """"
  DataAttribute: """"  (empty — acts on whole record)
  Use when: record cannot be corrected (e.g. missing lat without lon, orphaned record).

PredictDepthUsingIDW
  SaveType: Update
  RuleParameters: """"
  Use when: a depth-type field is missing or out of range on WellBore or MarkerPick.
  Mechanism: spatial Inverse Distance Weighting from neighboring wells that passed the same rule.
  Prerequisite: SURFACE_LATITUDE and SURFACE_LONGITUDE must be valid.

PredictFormationOrder
  SaveType: Update
  RuleParameters: """"
  Use when: formation age/order attribute is wrong or missing on MarkerPick.
  Mechanism: looks up STRAT_UNIT_ID → AGE from the formation pick dictionary.

PredictDominantLithology
  SaveType: Update
  RuleParameters: """"
  Use when: DOMINANT_LITHOLOGY is missing on MarkerPick.
  Mechanism: reads GR curve at PICK_DEPTH, smooths it, classifies lithology.

PredictLogDepthAttributes
  SaveType: Update
  RuleParameters: """"
  Use when: MIN_INDEX or MAX_INDEX is missing on Log.
  Mechanism: recalculates from the curve array itself — no external data needed.

PredictMissingDataObjects
  SaveType: Insert
  RuleParameters: {
    ""DataType"": ""<childType>"",
    ""Keys"": [
      {""Key"": ""<field>"", ""Value"": ""!<PARENT_FIELD>""},   ← !FIELD copies value from parent
      {""Key"": ""<field>"", ""Value"": ""<literal>""}
    ],
    ""Defaults"": [
      {""Default"": ""<field>"", ""Value"": ""<defaultValue>""}
    ]
  }
  Use when: an Entirety rule fails — a required child record is completely absent.
  Note: prefix a value with ! to copy that field from the parent data object.

════════════════════════════════════════════════════════
GENERATION RULES
════════════════════════════════════════════════════════

1. Generate exactly ONE rule per request — never a pair.
2. If the user asks for a DQ rule, generate only that DQ rule.
3. If the user asks for a prediction rule, generate only that prediction rule
   and set FailRule to the DQ rule key the user referenced.
4. Never generate both a DQ rule and a Prediction rule in the same response.
5. Only generate a Prediction rule when the user explicitly provides
   the FailRule key to link it to (e.g. "add a prediction for C4").
6. Set KeyNumber = 0 on all generated rules.
7. RuleKey values are temporary placeholders only.
8. Do not invent DataTypes, DataAttributes, or RuleFunctions.
9. Only use approved RuleFunctions from this prompt.
10. The response must be a valid JSON array containing exactly one RuleModel object.
    * No markdown, no explanations, no prose, no comments.
11. FIELD NAME RULES
- ALWAYS look up the DataType in the KNOWN SCHEMA section before choosing a DataAttribute.
- Use ONLY field names that appear in the schema for that DataType.
- NEVER guess, infer, or translate ANY plain English term into a field name using your own knowledge.
- For ANY attribute the user describes in plain English, find the closest matching field 
  in the KNOWN SCHEMA by scanning the full field list — do not rely on memory.
- If multiple fields could match, pick the one whose name most closely matches 
  the user's description and state your reasoning.
- If NO field in the schema matches the user's description, respond with a plain English 
  message saying which DataType was checked and that no matching field was found. 
  Do NOT generate a rule with an invented field name.
